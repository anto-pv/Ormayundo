◆ What it does

Two tools. Connected memory.

Ormayundo ships as a Claude plugin / MCP server exposing remember(text) and recall(query) — turning notes into a graph your agent can traverse.

01

Multi-hop recall

Retrieves connected facts, not just the nearest chunk — so “who worked with X on the project that shipped in Y?” actually resolves.

02

A graph, not flat search

An LLM extracts subject–relation–object facts into a knowledge graph of entities and relationships you can walk across.

03

Local & self-hosted

Stored in a local SQLite file. Embeddings run locally — your data, your infra. The only key you need is your LLM key.

◆ How it works

The pipeline

Quality comes from tuning the pipeline, not swapping the model. Text flows through five stages on its way into the graph and back out.

STAGE 01

Ingest

You call remember(text) with a note, transcript, or doc.

STAGE 02

Chunk

Text is split into coherent passages sized for extraction.

STAGE 03

Extract entities

An LLM pulls subject–relation–object facts from each chunk.

STAGE 04

Build graph

Facts merge into a SQLite-backed graph of entities and edges.

STAGE 05

Recall

recall(query) traverses edges to return connected facts.

◆ Get started

Up and running in three steps

Install, set your LLM key, and register the MCP server. Works with OpenAI by default, or Claude.

1

Install from PyPI.

shell 
$ pip install ormayundo
2

Export your LLM key. OpenAI is the default; switch to Claude with one env var.

shell 
# OpenAI (default)
export OPENAI_API_KEY=sk-...

# …or Claude instead
export LLM_PROVIDER=anthropic
export ANTHROPIC_API_KEY=sk-ant-...
3

Register the server in your .mcp.json.

.mcp.json 
{
  "mcpServers": {
    "ormayundo": {
      "command": "ormayundo-mcp"
    }
  }
}
→

Or drive it directly from Python.

example.py 
from ormayundo import remember, recall

remember("Priya led the Halley release with Arun in 2024.")

# multi-hop: traverses Priya → Halley → Arun
print(recall("who worked with Priya?"))

◆ Why a knowledge graph

Similar isn't the same as connected.

Vector search finds similar chunks. That's fine when the answer sits in one passage — but it can't hop from one fact to another.

A graph finds connected facts, which is exactly what multi-hop reasoning needs: follow an edge from one entity to the next until the chain resolves.

Based on the paper “Optimizing the Interface Between Knowledge Graphs and LLMs” (arXiv:2505.24478) — whose finding is that quality comes from tuning the pipeline, not the model.
